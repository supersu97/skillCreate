# 确认修改内容

本文件是 fullstack-development-helper 的确认模块。

## 步骤3：确认修改方案

**本步骤不再重复询问修改范围和类型（已在步骤2中收集），仅展示修改方案摘要供用户确认。**

### 生成修改方案摘要

根据步骤1（项目结构）、步骤1.5（代码风格）、步骤2（需求和修改方式）收集到的所有信息，生成修改方案摘要：

**只修改前端时**：

```
## 修改方案确认

### 基本信息
- 修改范围：前端
- 需求类型：[步骤2选择的需求类型]
- 修改方式：[步骤2选择的修改方式]

### 修改内容
[根据步骤2收集的详细需求，列出具体要修改的文件和内容]

### 涉及文件
1. [文件路径] - [修改说明]
2. [文件路径] - [修改说明]

### 代码风格
- 严格遵循步骤1.5中分析的前端代码风格

请确认以上修改方案是否正确？（确认后开始执行修改）
```

**只修改后端时**：

```
## 修改方案确认

### 基本信息
- 修改范围：后端
- 需求类型：[步骤2选择的需求类型]
- 修改方式：[步骤2选择的修改方式]

### 修改内容
[根据步骤2收集的详细需求，列出具体要修改的文件和内容]

### 涉及文件
1. [文件路径] - [修改说明]
2. [文件路径] - [修改说明]

### 数据库变更（如有）
[SQL语句或变更描述]

### 代码风格
- 严格遵循步骤1.5中分析的后端代码风格

请确认以上修改方案是否正确？（确认后开始执行修改）
```

**同时修改前后端时**：

```
## 修改方案确认

### 基本信息
- 修改范围：前端 + 后端
- 需求类型：[步骤2选择的需求类型]
- 前端修改方式：[步骤2选择的前端修改方式]
- 后端修改方式：[步骤2选择的后端修改方式]

### 前端修改内容
[根据步骤2收集的前端详细需求]

### 后端修改内容
[根据步骤2收集的后端详细需求]

### 涉及文件
**前端：**
1. [文件路径] - [修改说明]

**后端：**
1. [文件路径] - [修改说明]

### 数据库变更（如有）
[SQL语句或变更描述]

### 代码风格
- 前端：严格遵循步骤1.5中分析的前端代码风格
- 后端：严格遵循步骤1.5中分析的后端代码风格

请确认以上修改方案是否正确？（确认后开始执行修改）
```

**确认后**，根据步骤0的选择进入步骤4执行开发任务。
