# 前端代码执行器

本文件是 fullstack-development-helper 的前端执行模块，包含前端代码修改的详细执行流程。

## 步骤5：执行前端代码修改

**注意：修改方式已在步骤2中选择，本步骤直接根据已选择的修改方式执行，不再重复询问。**

### 根据修改方式执行对应流程

**方式1：直接描述修改**

1. 根据步骤2收集的修改描述，使用 Grep/Glob 工具定位相关文件
2. 使用 Read 工具读取文件内容
3. 分析代码，确定修改点
4. 使用 Edit/Write 工具执行修改

**方式2：根据接口信息修改**

1. 如果提供了接口路径：
   ```javascript
   调用 Grep 工具：
   - path: [前端项目路径]
   - pattern: [接口路径关键词]
   - glob: "**/*.{js,ts,vue,jsx,tsx}"
   ```

2. 如果提供了接口文档：
   - 读取并解析接口文档（参考 SKILL.md 接口文档处理流程）
   - 提取接口信息

3. 定位前端中对应的API调用代码
4. 使用 Read 工具读取相关文件
5. 根据接口信息修改前端代码

**方式3：指定组件或文件修改**

1. 使用 Glob 工具查找指定文件：
   ```javascript
   调用 Glob 工具：
   - path: [前端项目路径]/src
   - pattern: "**/*[组件名].vue" 或 "**/*[组件名].tsx"
   ```

2. 使用 Read 工具读取文件内容
3. 根据步骤2收集的修改描述执行修改

**方式4：新增组件或页面**

1. 根据步骤2收集的新增信息，确定文件位置和命名
2. 参考现有组件的代码风格（步骤1.5）
3. 创建新文件
4. 如需要，更新路由配置和导航

### 执行代码修改

1. 使用 Grep/Glob 工具定位需要修改的文件
2. 使用 Read 工具读取文件内容
3. 使用 Edit/Write 工具修改代码
4. 确认修改完成

### 重要约束

- 必须严格遵循步骤1.5中分析的前端代码风格
- 严格遵循用户指定的修改范围，没有要求改动的地方必须保持原样
- 修改完成后验证代码正确性
