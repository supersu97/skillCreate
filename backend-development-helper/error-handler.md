# 错误处理模块（新增）

本模块提供统一的错误处理和回滚机制，用于处理执行过程中的各种错误情况。

## 错误类型

### 1. 文件读取失败
- 文件不存在
- 权限不足
- 路径错误

### 2. 项目分析失败
- 无法识别项目类型
- 缺少关键配置文件
- 依赖解析失败

### 3. 代码修改失败
- 语法错误
- 类型错误
- 编译失败

### 4. Agent 执行失败
- 子 Agent 超时
- 子 Agent 返回错误
- 通信失败

## 处理流程

### 通用错误处理

当执行过程中出现错误时，按以下流程处理：

1. **记录错误信息**
   ```
   错误类型：[文件读取失败/项目分析失败/代码修改失败/Agent执行失败]
   错误位置：[步骤编号和具体操作]
   错误详情：[具体错误信息]
   上下文：[相关的文件路径、参数等]
   ```

2. **询问用户处理方式**

使用 AskUserQuestion 工具：

```javascript
{
  "questions": [{
    "question": "执行过程中出现错误，请选择处理方式：",
    "header": "错误处理",
    "multiSelect": false,
    "options": [
      {
        "label": "重试",
        "description": "使用相同参数重新执行"
      },
      {
        "label": "调整参数后重试",
        "description": "修改参数或路径后重新执行"
      },
      {
        "label": "跳过当前步骤",
        "description": "跳过当前步骤，继续执行后续步骤"
      },
      {
        "label": "终止执行",
        "description": "停止执行，保留已完成的修改"
      }
    ]
  }]
}
```

3. **根据用户选择执行对应操作**

**选择"重试"**：
- 使用相同参数重新执行失败的操作
- 最多重试 3 次
- 如果仍然失败，询问用户是否调整参数

**选择"调整参数后重试"**：
- 询问用户需要调整的参数
- 使用新参数重新执行
- 记录参数变更

**选择"跳过当前步骤"**：
- 记录跳过的步骤
- 继续执行后续步骤
- 在最终总结中提示用户跳过的步骤

**选择"终止执行"**：
- 停止执行
- 生成已完成的修改摘要
- 提示用户如何手动完成剩余工作

### Git 回滚机制（可选）

如果项目在 git 仓库中，可以提供回滚选项：

1. **修改前检查**
   ```javascript
   调用 Bash 工具：
   git rev-parse --is-inside-work-tree

   如果在 git 仓库中：
     调用 Bash 工具：
     git rev-parse HEAD
     记录当前 commit hash
   ```

2. **修改失败时询问是否回滚**

```javascript
{
  "questions": [{
    "question": "代码修改失败，是否回滚到修改前的状态？",
    "header": "回滚确认",
    "multiSelect": false,
    "options": [
      {
        "label": "回滚",
        "description": "使用 git reset 回滚到修改前的状态"
      },
      {
        "label": "保留修改",
        "description": "保留已完成的修改，手动处理错误"
      }
    ]
  }]
}
```

3. **执行回滚**

如果用户选择回滚：

```javascript
调用 Bash 工具：
git reset --hard [记录的 commit hash]

提示用户：
"已回滚到修改前的状态（commit: [hash]）"
```

## 错误预防

### 1. 文件操作前验证
- 检查文件是否存在
- 检查路径是否正确
- 检查权限是否足够

### 2. 代码修改前备份
- 如果在 git 仓库中，记录当前 commit
- 如果不在 git 仓库中，建议用户先提交到版本控制

### 3. 分步执行
- 将大任务拆分为小步骤
- 每个步骤完成后验证
- 出错时只需回滚当前步骤

### 4. 提供详细的错误信息
- 不要只显示"失败"
- 提供具体的错误原因
- 提供可能的解决方案

## 示例

### 示例 1：文件读取失败

```
错误类型：文件读取失败
错误位置：步骤 3.2 - 分析项目结构
错误详情：无法读取文件 /path/to/package.json
上下文：前端项目路径 = /path/to/project

可能原因：
1. 文件不存在
2. 路径错误
3. 权限不足

建议操作：
1. 检查项目路径是否正确
2. 检查文件是否存在
3. 使用 "调整参数后重试" 选项，提供正确的项目路径
```

### 示例 2：代码修改失败

```
错误类型：代码修改失败
错误位置：步骤 7 - 执行前端代码修改
错误详情：语法错误 - Unexpected token '}'
上下文：修改文件 = src/components/UserList.vue

可能原因：
1. 代码格式错误
2. 缺少必要的导入
3. 类型定义错误

建议操作：
1. 使用 "重试" 选项，重新生成代码
2. 使用 "调整参数后重试" 选项，提供更详细的修改要求
3. 如果在 git 仓库中，可以选择回滚
```
