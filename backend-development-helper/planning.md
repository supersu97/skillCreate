# 后端规划模块

本模块用于后端开发任务的规划，包含轻量规划和深度规划。

## 适用场景

**简单任务（跳过规划）**：
- 修改 1-2 个文件
- Bug 修复
- 简单字段调整

**中等任务（轻量规划）**：
- 修改 3-5 个文件
- 修改现有 API 逻辑
- 小范围优化

**复杂任务（深度规划）**：
- 新增功能模块
- 修改 6+ 个文件
- 涉及数据库变更
- 新增完整的 API 接口链路

## 轻量规划流程（中等任务）

### 1. 明确功能边界和目标

```markdown
## 功能边界和目标

### 当前任务范围
- 修改内容：[具体要修改的功能点]
- 影响范围：[后端/数据库]
- 不包含内容：[明确不在本次修改范围内的内容]

### 预期目标
- 功能目标：[要实现的具体效果]
- 性能目标：[如有性能要求，明确指标]

### 成功标准
- [ ] [标准 1]
- [ ] [标准 2]
```

### 2. 技术约束和规范

```markdown
## 后端技术约束（必须遵守）

- 框架版本：[如：Spring Boot 3.1.x]
- 数据库：[如：MySQL 8.0]
- ORM：[如：JPA/Hibernate]
- 代码风格：[引用步骤 3.3 的后端代码风格]
- 禁止使用：[如：不允许使用原生 JDBC]
```

### 3. 交付物清单

```markdown
## 交付物清单

### 代码文件
- [ ] [文件路径 1]：[修改内容说明]
- [ ] [文件路径 2]：[修改内容说明]

### 数据库（如需要）
- [ ] SQL 迁移脚本

### 文档（如需要）
- [ ] 接口文档更新
```

### 4. 实施计划

```markdown
## 实施计划

### 第一阶段：准备工作
- [ ] 定位需要修改的文件
- [ ] 理解现有代码逻辑
- [ ] 确认依赖关系

### 第二阶段：代码修改
- [ ] [具体修改步骤 1]
- [ ] [具体修改步骤 2]

### 第三阶段：验证和测试
- [ ] 代码风格检查
- [ ] 功能测试
- [ ] 回归测试
```

## 深度规划流程（复杂任务）

### 1. 进入 Plan Mode

使用 EnterPlanMode 工具进入规划模式：

```javascript
调用 EnterPlanMode 工具
```

### 2. 需求分析与模块划分

在 Plan Mode 中执行详细的需求分析：
- 分析业务需求
- 划分功能模块
- 确定模块间依赖关系

### 3. 技术方案设计

为每个模块确定实现思路：
- 数据库表设计
- API 接口设计
- 业务逻辑分层
- 技术选型

### 4. 任务优先级排序

根据依赖关系和重要性确定实现顺序。

### 5. 生成实施计划文档

将所有内容整合为一份完整的实施计划文档，保存到项目目录：

```javascript
调用 Write 工具：
- file_path: [后端项目路径]/docs/implementation-plan-[YYYYMMDD-HHmmss].md
- content: [整合所有规划内容]
```

### 6. 退出 Plan Mode 并用户确认

使用 ExitPlanMode 工具退出规划模式，并将实施计划提交给用户审核。
