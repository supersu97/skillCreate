# 前端规划模块

本模块用于前端开发任务的规划，包含轻量规划和深度规划。

## 适用场景

**简单任务（跳过规划）**：
- 修改 1-2 个文件
- Bug 修复
- 样式调整

**中等任务（轻量规划）**：
- 修改 3-5 个文件
- 修改现有功能
- 小范围优化

**复杂任务（深度规划）**：
- 新增功能模块
- 修改 6+ 个文件
- 新增页面/复杂组件

## 轻量规划流程（中等任务）

### 1. 明确功能边界和目标

```markdown
## 功能边界和目标

### 当前任务范围
- 修改内容：[具体要修改的功能点]
- 影响范围：[前端]
- 不包含内容：[明确不在本次修改范围内的内容]

### 预期目标
- 功能目标：[要实现的具体效果]
- 用户体验目标：[如有 UX 要求，明确标准]

### 成功标准
- [ ] [标准 1]
- [ ] [标准 2]
```

### 2. 技术约束和规范

```markdown
## 前端技术约束（必须遵守）

- 框架版本：[如：Vue 3.2.x]
- UI 组件库：[如：Element Plus 2.x]
- 状态管理：[如：Pinia]
- 代码风格：[引用步骤 3.3 的前端代码风格]
- 禁止使用：[如：不允许使用 jQuery]

### 设计规范约束
- 颜色规范：[引用步骤 3.3 的设计规范]
- 间距规范：[引用步骤 3.3 的间距规范]
- 字体规范：[引用步骤 3.3 的字体规范]
```

### 3. 交付物清单

```markdown
## 交付物清单

### 代码文件
- [ ] [文件路径 1]：[修改内容说明]
- [ ] [文件路径 2]：[修改内容说明]

### 文档（如需要）
- [ ] README 更新
```

### 4. 实施计划

```markdown
## 实施计划

### 第一阶段：准备工作
- [ ] 定位需要修改的文件
- [ ] 理解现有代码逻辑
- [ ] 确认依赖关系

### 第二阶段：代码修改
- [ ] [具体修改步骤 1]
- [ ] [具体修改步骤 2]

### 第三阶段：验证和测试
- [ ] 代码风格检查
- [ ] 功能测试
- [ ] 浏览器兼容性测试
```

## 深度规划流程（复杂任务）

### 1. 进入 Plan Mode

使用 EnterPlanMode 工具进入规划模式：

```javascript
调用 EnterPlanMode 工具
```

### 2. 需求分析与模块划分

在 Plan Mode 中执行详细的需求分析：
- 分析 UI/UX 需求
- 划分组件模块
- 确定组件间通信方式

### 3. 技术方案设计

为每个模块确定实现思路：
- 组件结构设计
- 状态管理方案
- API 对接方式
- 样式实现方案

### 4. 任务优先级排序

根据依赖关系和重要性确定实现顺序。

### 5. 生成实施计划文档

将所有内容整合为一份完整的实施计划文档，保存到项目目录：

```javascript
调用 Write 工具：
- file_path: [前端项目路径]/docs/implementation-plan-[YYYYMMDD-HHmmss].md
- content: [整合所有规划内容]
```

### 6. 退出 Plan Mode 并用户确认

使用 ExitPlanMode 工具退出规划模式，并将实施计划提交给用户审核。
