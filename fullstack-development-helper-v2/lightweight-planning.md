# 轻量规划模块

本模块用于修改现有功能、bug 修复、小范围优化等中等复杂度任务的规划。

## 适用场景

- 修改现有功能
- Bug 修复
- 小范围优化
- 修改 3-5 个文件

## 规划流程

### 1. 明确功能边界和目标

基于步骤 4 收集的需求，生成功能边界文档：

```markdown
## 功能边界和目标

### 当前任务范围
- 修改内容：[具体要修改的功能点]
- 影响范围：[前端/后端/数据库]
- 不包含内容：[明确不在本次修改范围内的内容]

### 预期目标
- 功能目标：[要实现的具体效果]
- 性能目标：[如有性能要求，明确指标]
- 用户体验目标：[如有 UX 要求，明确标准]

### 成功标准
- [ ] [标准 1：如"用户可以成功提交表单"]
- [ ] [标准 2：如"响应时间 < 500ms"]
- [ ] [标准 3：如"兼容 Chrome/Firefox/Safari"]
```

### 2. 技术约束和规范

基于步骤 3.3 分析的代码风格，生成技术约束文档：

```markdown
## 技术约束和规范（必须遵守）

### 前端技术约束（如涉及前端）
- 框架版本：[如：Vue 3.2.x]
- UI 组件库：[如：Element Plus 2.x]
- 状态管理：[如：Pinia]
- 代码风格：[引用步骤 3.3 的前端代码风格]
- 禁止使用：[如：不允许使用 jQuery、不允许直接操作 DOM]

### 后端技术约束（如涉及后端）
- 框架版本：[如：Spring Boot 3.1.x]
- 数据库：[如：MySQL 8.0]
- ORM：[如：JPA/Hibernate]
- 代码风格：[引用步骤 3.3 的后端代码风格]
- 禁止使用：[如：不允许使用原生 JDBC、不允许使用 XML 配置]

### 设计规范约束（如涉及前端 UI）
- 颜色规范：[引用步骤 3.3 的设计规范]
- 间距规范：[引用步骤 3.3 的间距规范]
- 字体规范：[引用步骤 3.3 的字体规范]
- 响应式断点：[引用步骤 3.3 的断点规范]
```

### 3. 期望的输出格式和交付物

```markdown
## 交付物清单

### 代码文件
- [ ] [文件路径 1]：[修改内容说明]
- [ ] [文件路径 2]：[修改内容说明]

### 文档（如需要）
- [ ] 接口文档更新（如有接口变更）
- [ ] README 更新（如有使用方式变更）

### 测试（如需要）
- [ ] 单元测试（如有新增函数）
- [ ] 集成测试（如有接口变更）
```

### 4. 实施计划

```markdown
## 实施计划

### 第一阶段：准备工作
- [ ] 定位需要修改的文件
- [ ] 理解现有代码逻辑
- [ ] 确认依赖关系

### 第二阶段：代码修改
- [ ] [具体修改步骤 1]
- [ ] [具体修改步骤 2]
- [ ] [具体修改步骤 3]

### 第三阶段：验证和测试
- [ ] 代码风格检查
- [ ] 功能测试
- [ ] 回归测试（确保未破坏现有功能）
```

### 5. 用户确认

将上述四部分内容整合后，使用 AskUserQuestion 工具让用户确认：

```javascript
{
  "questions": [{
    "question": "请确认以上规划是否符合预期？",
    "header": "规划确认",
    "multiSelect": false,
    "options": [
      {
        "label": "确认，开始执行",
        "description": "规划符合预期，可以开始代码修改"
      },
      {
        "label": "需要调整",
        "description": "规划需要修改，请说明需要调整的地方"
      }
    ]
  }]
}
```

如果用户选择"需要调整"，根据用户反馈修改规划，重新确认。
