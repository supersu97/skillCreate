# 需求收集器（优化版）

本文件是 fullstack-development-helper-v2 的需求收集模块，包含如何收集开发需求的详细流程。

## v2 优化说明

相比 v1 版本：
- ✅ 基于步骤 1 的智能推断结果，只询问必要的补充信息
- ✅ 减少重复询问，提高效率
- ✅ 自动判断是否需要 UI 设计图处理

## 步骤 4：收集开发需求

**本步骤基于步骤 1 的推断结果，只收集必要的补充信息**

### 4.1 根据推断的修改范围收集详细需求

#### 只修改前端时

基于步骤 1 推断的修改方式，收集对应的详细信息：

**修改方式为"直接描述修改"时**：

```
请详细描述要修改的内容：
- 修改目标：
- 修改要求：
- 预期效果：
```

**修改方式为"根据接口信息修改"时**：

```
请提供接口信息（二选一）：
1. 接口路径和请求方法（如：GET /api/users）
2. 接口文档位置（本地文件路径或在线链接）
```

**修改方式为"指定文件修改"时**：

```
请提供要修改的文件信息：
- 文件路径或名称（如：src/components/UserList.vue）
- 修改内容描述：
```

**修改方式为"新增组件/页面"时**：

```
请提供新增内容信息：
- 名称：
- 位置：
- 功能描述：
- 包含的内容和交互逻辑：
```

**如果是前端修改且涉及UI/样式变更，额外收集**：

使用 AskUserQuestion 询问：
```javascript
{
  "questions": [{
    "question": "是否有UI设计图或样式要求？",
    "header": "UI设计",
    "multiSelect": false,
    "options": [
      {
        "label": "有设计图",
        "description": "提供设计图文件路径或截图"
      },
      {
        "label": "有样式描述",
        "description": "文字描述样式要求（颜色、布局、尺寸等）"
      },
      {
        "label": "参考现有组件",
        "description": "参考项目中已有的组件样式"
      },
      {
        "label": "无特殊要求",
        "description": "使用项目默认样式规范"
      }
    ]
  }]
}
```

**如果用户选择"有设计图"**：
```
必须使用 Read 工具读取 ui-design-handler.md 文件，然后严格按照文件中的流程执行 UI 设计图处理。

读取命令：
调用 Read 工具：
- file_path: [技能基础路径]/ui-design-handler.md

执行 ui-design-handler.md 中的所有步骤，提取设计图中的布局、颜色、尺寸等信息。
```

#### 只修改后端时

基于步骤 1 推断的修改方式，收集对应的详细信息：

**修改方式为"直接描述修改"时**：

```
请详细描述要修改的内容：
- 修改目标：
- 修改要求：
- 预期效果：
```

**修改方式为"根据接口信息修改"时**：

```
请提供接口信息（二选一）：
1. 接口路径和请求方法（如：POST /api/users）
2. 接口文档位置（本地文件路径或在线链接）
```

**修改方式为"指定文件修改"时**：

```
请提供要修改的文件信息：
- 文件路径或类名（如：UserController 或 src/main/java/.../UserService.java）
- 修改内容描述：
```

**修改方式为"新增API/功能"时**：

```
请提供新增内容信息：
- API 名称：
- 接口路径：
- 功能描述：
- 请求参数和响应格式：
```

**询问是否需要数据库变更**：

```javascript
{
  "questions": [{
    "question": "是否需要数据库变更？",
    "header": "数据库",
    "multiSelect": false,
    "options": [
      {
        "label": "需要",
        "description": "需要修改数据库表结构，将生成相应SQL语句"
      },
      {
        "label": "不需要",
        "description": "不涉及数据库变更"
      }
    ]
  }]
}
```

**如果选择了"需要数据库变更"**：

```
请描述数据库变更内容：
- 变更类型（新增表/修改表/新增字段等）：
- 变更描述或SQL语句：
```

#### 同时修改前后端时

分别收集前端和后端的详细需求（使用上述逻辑）。

**前端需求**：
- 基于步骤 1 推断的前端修改方式收集
- 如果涉及 UI，询问设计图

**后端需求**：
- 基于步骤 1 推断的后端修改方式收集
- 询问是否需要数据库变更

**询问是否需要对接第三方接口**：

```javascript
{
  "questions": [{
    "question": "后端是否需要对接第三方接口？",
    "header": "第三方接口",
    "multiSelect": false,
    "options": [
      {
        "label": "需要对接第三方接口",
        "description": "需要调用第三方 API，请提供接口文档路径或链接"
      },
      {
        "label": "不需要对接第三方接口",
        "description": "仅修改内部逻辑，根据需求自行设计接口"
      }
    ]
  }]
}
```

如果需要对接第三方接口，收集接口文档路径后解析。

### 4.2 生成需求摘要

将收集到的所有信息整理为需求摘要：

```
## 需求摘要

### 基本信息
- 修改范围：[前端/后端/全栈]
- 需求类型：[新增功能/修改现有功能/Bug修复/性能优化]
- 修改方式：[具体方式]

### 详细需求
[根据收集到的信息填充]

### 涉及内容
- 前端：[如有]
- 后端：[如有]
- 数据库：[如有]
- 第三方接口：[如有]
- UI设计：[如有]
```

**记录所有收集到的信息**，传递给后续步骤使用。
